# Лабораторная работа № 5

## Обзор проекта
Целью данной работы была реализация гибкой системы обработки данных на языке C++ с использованием **указателей на функции**. Архитектура программы построена на принципе отделения общего алгоритма обработки от конкретных правил фильтрации и преобразования.

## Выполненные задачи

### 1. Реализация стратегий (Вариант №9)
Для выполнения задания по варианту были разработаны две "чистые" функции:
* **Фильтрация (filter)**: Выбирает только те числа, которые кратны 4 (`value % 4 == 0`).
* **Преобразование (transform)**: Удваивает значение выбранных элементов (`value * 2`).

### 2. Разработка основного алгоритма
Была написана универсальная функция `process`, которая выполняет следующие шаги:
* **Валидация**: Проверка входного массива на пустоту и корректность размера.
* **Подсчет**: Первый проход по массиву для определения количества элементов, подходящих под фильтр.
* **Управление памятью**: Динамическое выделение массива нужного размера с помощью оператора `new[]`.
* **Обработка**: Второй проход для заполнения нового массива результатами трансформации.

### 3. Модульное тестирование
Для подтверждения надежности кода были созданы тестовые сценарии без использования сторонних фреймворков:
* **Стандартный случай**: Проверка того, что массив корректно преобразуется.
* **Граничные условия**: Проверка поведения системы при **пустом входном массиве** и в ситуациях, когда **ни один элемент не подходит** под условие фильтра.
* **Безопасность памяти**: Гарантированное освобождение динамической памяти с помощью `delete[]` во всех тестах для предотвращения утечек.

## Заключение
Работа демонстрирует преимущества функционального подхода в C++. Благодаря передаче функций в качестве аргументов, логика `process` остается неизменной, даже если правила фильтрации или трансформации изменятся.
